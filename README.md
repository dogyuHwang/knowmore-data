# knowmore-data

아만보 앱의 카드 콘텐츠 데이터 레포입니다.
앱 업데이트 없이 이 레포만 수정하면 최대 2시간 내 사용자에게 반영됩니다.

이 README는 Claude가 데이터 업데이트 요청을 받았을 때 따라야 할 기준과 절차를 정의합니다.
**데이터 업데이트 요청 시 반드시 이 문서 전체를 먼저 읽고 작업하세요.**

---

## 파일 구조

```
knowmore-data/
├── hot_issues.json   # 이슈 🔥 (매주 업데이트)
├── economy.json      # 경제
├── stock.json        # 주식
├── ai.json           # AI
├── it.json           # IT
├── law.json          # 법률
├── health.json       # 건강
└── real_estate.json  # 부동산
```

## 카드 JSON 형식

각 파일은 카드 객체의 **배열**입니다.

```json
[
  {
    "id": "ai_001",
    "topic": "AI",
    "word": "인공지능",
    "summary": "인간의 지능을 모방해 학습·판단·예측할 수 있도록 만든 컴퓨터 시스템",
    "description": "인공지능(AI)은 컴퓨터가 스스로 학습하고 문제를 해결할 수 있도록 만든 기술이에요. 얼굴 인식, 번역, 추천 알고리즘 등 일상 곳곳에 이미 쓰이고 있어요.",
    "example": "유튜브가 내 취향에 맞는 영상을 추천해주는 것도 인공지능 알고리즘 덕분이에요.",
    "level": "뉴비",
    "active": true
  }
]
```

### 필드 설명

| 필드 | 필수 | 설명 |
|------|------|------|
| `id` | ✅ | 고유 식별자. `{파일명}_{번호}` 형식 (예: `ai_001`) |
| `topic` | ✅ | 주제 이름. 파일별 고정값 사용 (아래 표 참고) |
| `word` | ✅ | 카드 제목 (단어/용어) |
| `summary` | ✅ | 한 줄 요약 (40자 이내 권장) |
| `description` | ✅ | 본문 설명 (3~5문장, 구어체 ~에요 말투) |
| `example` | ✅ | 실생활 예시 문장 (구어체, 대화체 권장) |
| `level` | ✅ | 난이도: `뉴비` / `고인물` / `신` |
| `active` | ❌ | 생략 시 `true`. `false`면 앱에서 숨김 |
| `updatedAt` | ❌ | 이슈 분야 전용. 업데이트 날짜 `YYYY-MM-DD` 형식 |

### 파일별 topic 값

| 파일 | topic 값 |
|------|----------|
| `hot_issues.json` | `이슈` |
| `economy.json` | `경제` |
| `stock.json` | `주식` |
| `ai.json` | `AI` |
| `it.json` | `IT` |
| `law.json` | `법률` |
| `health.json` | `건강` |
| `real_estate.json` | `부동산` |

## 난이도 기준

앱은 뉴비를 전부 학습해야 고인물이 해금되고, 고인물을 전부 학습해야 신이 해금되는 구조입니다.
카드를 추가하면 해당 난이도의 미학습 카드가 늘어나 기존 사용자의 해금 진행도가 초기화될 수 있으니 신중하게 추가하세요.

| 값 | 해금 조건 | 기준 |
|----|----------|------|
| `뉴비` | 처음부터 열림 | 뉴스에서 자주 나오는 기본 개념. 처음 들어도 이해되는 수준 |
| `고인물` | 뉴비 전체 학습 후 | 배경 지식이 있어야 이해되는 내용. 구조나 원인까지 설명 |
| `신` | 고인물 전체 학습 후 | 전문가 수준의 심화 개념. 세부 메커니즘이나 파생 개념 |

### 난이도 판단 기준 예시

- **뉴비**: "인플레이션", "GPT", "전세", "탄핵" — 뉴스 제목에 자주 등장, 설명 없이도 들어본 단어
- **고인물**: "테이퍼링", "RAG", "역전세", "헌법소원" — 뉴스 본문에서 다루는 개념, 배경 지식 필요
- **신**: "양적완화 출구전략", "파인튜닝 vs RLHF", "선순위 근저당", "위헌법률심판 제청" — 전문가들이 쓰는 세부 용어

## 일반 주제 업데이트 규칙 (hot_issues 제외)

### 추가 기준
- 주제당 총 카드 수 권장: **뉴비 8개 / 고인물 8개 / 신 4개** (총 20개)
- 최대 **뉴비 16개 / 고인물 16개 / 신 8개**를 넘지 않도록 유지
- 너무 유사한 카드(비슷한 개념을 다루는 경우)는 추가하지 않음
- 새 카드 추가 시 `id`는 기존 마지막 번호 다음부터 순차 부여

### 삭제/비활성화 기준
다음에 해당하는 카드는 `active: false` 처리하거나 삭제합니다.

- 현재 시점에서 더 이상 유효하지 않은 정보 (폐지된 제도, 사라진 서비스 등)
- 수치나 통계가 크게 달라진 경우 (예: 금리, 지수 등 — 업데이트 또는 삭제)
- 이미 해결된 일시적 이슈 (ex. 특정 사건·스캔들)
- 더 좋은 카드로 대체 가능한 중복 개념

### ID 관리
- 삭제된 카드의 `id`는 재사용하지 않습니다.
  (앱 DB에 해당 cardId로 학습 기록이 남아 있을 수 있어 충돌 방지)
- 새 카드는 항상 기존 최대 번호 + 1부터 시작합니다.

## 이슈 분야 업데이트 규칙 (hot_issues.json)

이슈 분야는 **매주 전면 교체** 방식으로 운영합니다.

### 주제 선정 기준
- 업데이트 시점 기준 **최근 1~2주 내** 한국 사회에서 크게 거론된 이슈
- 대상 영역: 경제·부동산·주식·사회·국제·IT·AI·건강 등 (정치 주제는 제외)
- 일시적 가십이 아닌, 구조적 배경이 있어 설명할 가치가 있는 주제

### 카드 수 제한
- 총 카드 수: **최소 20개, 최대 60개**
- **기존에 20개가 있어도 이슈가 충분하면 30~60개까지 적극적으로 추가** — 최소치에 맞추지 말 것
- 다룰 이슈가 많을수록 카드를 더 많이 작성하는 것이 원칙
- 난이도 배분은 이슈의 특성에 따라 자유롭게 판단
  (해당 이슈가 대중에게 생소하면 뉴비를 더 많이, 이미 널리 알려진 이슈면 고인물·신 비중을 높임)

### updatedAt 처리
- 모든 카드의 `updatedAt`은 **업데이트 당일 날짜**(`YYYY-MM-DD`)로 설정
- 앱에서 카드 상세 화면과 홈 화면에 업데이트 날짜가 표시됨

### ID 관리
- 이슈 분야는 매주 전면 교체이므로 `hot_issues_001`부터 순차 재사용 가능
- 카드 수가 변경되면 초과하는 기존 번호는 자동으로 사라짐

### 예시

```json
{
  "id": "hot_issues_001",
  "topic": "이슈",
  "word": "트럼프 관세",
  "summary": "미국이 주요국에 고율 관세를 부과하며 무역 갈등이 심화됐어요",
  "description": "...",
  "example": "...",
  "level": "뉴비",
  "updatedAt": "2026-03-26"
}
```

## 콘텐츠 작성 가이드

### 문체
- `description`: 구어체, ~에요/~이에요 말투. 3~5문장. 개념 설명 → 배경 → 한국과의 연관성 순서 권장
- `summary`: 명사형 또는 ~했어요 말투. 40자 이내
- `example`: 실제 대화나 일상에서 나올 법한 문장. "~라고 하더라고요", "~이 걱정돼요" 같은 자연스러운 구어체

### 금지 사항
- 특정 정치인·기업·개인에 대한 주관적 평가
- 확인되지 않은 정보나 추측
- 지나치게 전문적인 영어 약어만 나열 (반드시 한국어 풀이 포함)
- 동일한 개념을 다른 단어로 중복 등재

### 앱 UI 고려
- `word` 길이: 10자 이내 권장 (홈 카드 타이틀에 표시)
- `summary` 길이: 40자 이내 (카드 미리보기에 표시)
- `description` 줄 수: 너무 길면 스크롤이 길어지므로 5문장 이내 권장

## 업데이트 절차

### 이슈 분야 주간 업데이트 요청 시

```
knowmore-data README를 참고해서 hot_issues.json을 이번 주 이슈로 업데이트해줘.
오늘 날짜: YYYY-MM-DD
```

Claude가 수행하는 작업:
1. 최근 1~2주 주요 이슈 탐색 (웹 검색 활용)
2. 주제 선정 및 난이도 배분
3. 카드 작성 (오늘 날짜로 `updatedAt` 설정)
4. `hot_issues.json` 교체
5. GitHub push

### 일반 주제 업데이트 요청 시

```
[파일명].json을 최신화해줘. / [단어들]을 추가해줘. / 오래된 카드 정리해줘.
```

Claude가 수행하는 작업:
1. 기존 카드 검토 (삭제/비활성화 대상 식별)
2. 추가할 카드 작성 (난이도 자동 판단)
3. ID 연속성 유지하며 추가
4. GitHub push

## 주의사항

- `id`는 전체 데이터에서 **유일**해야 합니다. 중복 시 앱 오동작 가능
- 삭제보다 `active: false` 처리를 권장합니다 (사용자 DB에 학습 기록이 남아 있을 수 있음)
- 앱은 **2시간마다** GitHub에서 새 데이터를 fetch합니다
- `뉴비` 카드를 추가하면 해당 주제의 해금 진행도가 초기화될 수 있으니 불필요한 추가는 자제합니다
- 카드를 삭제해도 사용자 DB의 학습 기록(`cardHistories`)은 남아 있으나, 앱 로직상 문제없이 동작합니다
